4.1 Konsep Dasar Model 1PL

Model 1 Parameter Logistik (1PL) adalah model IRT paling sederhana. Disebut “1 parameter” karena hanya satu parameter item yang diestimasi, yaitu parameter kesulitan (b).

Model ini mengasumsikan bahwa:

  • Semua item memiliki daya beda yang sama (a konstan = 1.0)
  • Tidak ada efek tebakan (c = 0)
  • Hanya posisi kurva pada skala θ yang berbeda antar item

4.2 Formula Model 1PL

\[P(X_{ij} = 1 \mid \theta_j, b_i) = \frac{e^{(\theta_j - b_i)}}{1 + e^{(\theta_j - b_i)}}\]

Yang equivalen dengan:

\[P(X_{ij} = 1 \mid \theta_j, b_i) = \frac{1}{1 + e^{-(\theta_j - b_i)}}\]

Di mana:

  • \(\theta_j\) = kemampuan peserta \(j\) (dalam logit)
  • \(b_i\) = parameter kesulitan item \(i\) (dalam logit)
  • \(P(\cdot)\) = probabilitas menjawab benar
NoteHubungan 1PL dengan Rasch Model

Model 1PL dan Rasch Model secara matematis identik ketika konstanta D = 1. Perbedaan filosofis ada pada pendekatan:

1PL (dalam IRT) Rasch Model
Orientasi Fit model ke data Data harus fit ke model
Estimasi MML (mirt) CML (eRm) atau JML (TAM)
Perlakuan misfit Item direvisi jika perlu Item harus fit ke model

4.3 ICC itu apa?

ICC = Item Characteristic Curve — kurva yang menggambarkan hubungan antara kemampuan seseorang dengan peluang dia menjawab benar satu soal tertentu.

Bentuknya selalu S (sigmoid) — landai di ujung kiri, naik curam di tengah, landai lagi di ujung kanan.

library(ggplot2)

theta <- seq(-4, 4, by = 0.05)

icc_1pl <- function(theta, b) {
  1 / (1 + exp(-(theta - b)))

4.5 Item Information Function (IIF) — Model 1PL

Untuk model 1PL, IIF memiliki bentuk:

\[I_i(\theta) = P_i(\theta) \cdot [1 - P_i(\theta)]\]

Informasi maksimum terjadi ketika \(\theta = b\), yaitu:

\[I_i^{max} = 0.5 \times 0.5 = 0.25\]

ImportantImplikasi Informasi Maksimum

Karena semua item 1PL memiliki a = 1, semua item memberikan informasi maksimum yang sama (0.25) — hanya berbeda di mana informasi tersebut paling tinggi (yaitu pada θ = b masing-masing item).

Code 
iif_1pl <- function(theta, b) {
  p <- icc_1pl(theta, b)
  p * (1 - p)
}

df_iif <- data.frame(
  theta = rep(theta, length(b_values)),
  info  = unlist(lapply(b_values, function(b) iif_1pl(theta, b))),
  item  = factor(rep(labels, each = length(theta)), levels = labels)

Yang Berbeda: Posisi Puncaknya

Kurva b Puncak informasi di Artinya
🔵 Biru tua -2 θ = -2 Paling berguna untuk peserta berkemampuan sangat rendah
🩵 Biru muda -1 θ = -1 Paling berguna untuk peserta di bawah rata-rata
🟡 Kuning 0 θ = 0 Paling berguna untuk peserta rata-rata
🟠 Oranye +1 θ = +1 Paling berguna untuk peserta di atas rata-rata
🔴 Merah +2 θ = +2 Paling berguna untuk peserta berkemampuan sangat tinggi

Polanya jelas: Puncak informasi selalu tepat berada di θ = b.

Analogi: Lampu Sorot di Panggung

Penonton duduk dari kiri (-4) ke kanan (+4)

Setiap lampu menerangi SATU titik paling terang
tapi intensitas maksimalnya SAMA semua

Tidak ada lampu yang “lebih kuat” dari yang lain — hanya berbeda posisi yang diteranginya.

ImportantApa yang Ditunjukkan Grafik Ini?

Grafik Ini Menguji Apa?

Parameter Recovery adalah uji validasi fundamental dalam IRT:

“Jika kita tahu nilai b yang sebenarnya (true value), seberapa akurat metode estimasi berhasil menemukannya kembali?”

Ini seperti ujian bagi metode estimasinya — bukan ujian bagi peserta.

Cara Membaca Grafik

Sumbu X = nilai b yang sesungguhnya (true value, kita yang tentukan saat simulasi)

Sumbu Y = nilai b yang diestimasi oleh software

Garis putus-putus = garis sempurna (estimasi = true value persis)

Prinsipnya sederhana:
>Titik makin dekat ke garis putus-putus = estimasi makin akurat

>Titik jauh dari garis = estimasi meleset (bias)

Apa yang Bisa Disimpulkan?

  1. Kedua metode sangat akurat
  • Semua titik — baik lingkaran maupun segitiga — menempel sangat dekat pada garis putus-putus. Tidak ada yang menyimpang jauh. Ini menunjukkan kedua metode berhasil me-recover parameter b dengan baik.
  1. Kedua metode menghasilkan estimasi yang hampir identik
  • Titik biru (CML) dan segitiga oranye (MML) saling tumpang tindih di sepanjang garis. Artinya dalam konteks Model 1PL, pilihan metode estimasi tidak banyak membuat perbedaan hasil.
  1. Akurasi konsisten di seluruh rentang b
  • Dari b = -2.5 hingga b = +2.5, tidak ada area yang estimasinya lebih buruk. Artinya model 1PL bekerja stabil baik untuk soal mudah maupun soal sulit.

4.7 Item Fit — Model 1PL

# Estimasi item fit
fit_1pl <- mirt::itemfit(model_1pl)

# Ambil p-value; clamp ke [1e-6, 1] agar -log10() tidak menghasilkan Inf/NaN
p_raw <- fit_1pl$p.S_X2
p_raw <- pmax(p_raw, 1e-6)          # hindari -log10(0) = Inf
p_raw <- pmin(p_raw, 1)             # pastikan tidak > 1

fit_df <- data.frame(
  Item      = paste0("Item", sprintf("%02d", 1:n_item)),

Interpretasi Item Fit S-X² — Model 1PL
Kesimpulan Utama: Semua Item FIT ✅

Seluruh 20 item berwarna hijau dan batangnya berada di sebelah kiri garis merah putus-putus (batas p < .05). Artinya tidak ada satu pun item yang misfit terhadap Model 1PL.

Cara Membaca Grafik Ini

Batang melewati garis merah → item MISFIT (harus dievaluasi)
Batang tidak melewati garis → item FIT ✅

4.7.2 Apa Artinya “Fit” dalam Konteks Ini?

Item dikatakan fit terhadap Model 1PL jika pola respons peserta pada item tersebut konsisten dengan prediksi model. Secara konkret:

Peserta berkemampuan tinggi memang lebih banyak menjawab benar, dan peserta berkemampuan rendah lebih banyak menjawab salah — sesuai dengan yang diprediksi kurva ICC model 1PL.

# Model 1 Parameter Logistik (1PL)

## Konsep Dasar Model 1PL

**Model 1 Parameter Logistik (1PL)** adalah model IRT paling sederhana. Disebut "1 parameter" karena hanya satu parameter item yang diestimasi, yaitu **parameter kesulitan (*b*)**.

Model ini mengasumsikan bahwa:

-   Semua item memiliki **daya beda yang sama** (*a* konstan = 1.0)
-   Tidak ada efek tebakan (*c* = 0)
-   Hanya posisi kurva pada skala θ yang berbeda antar item

## Formula Model 1PL

$$P(X_{ij} = 1 \mid \theta_j, b_i) = \frac{e^{(\theta_j - b_i)}}{1 + e^{(\theta_j - b_i)}}$$

Yang equivalen dengan:

$$P(X_{ij} = 1 \mid \theta_j, b_i) = \frac{1}{1 + e^{-(\theta_j - b_i)}}$$

Di mana:

-   $\theta_j$ = kemampuan peserta $j$ (dalam logit)
-   $b_i$ = parameter kesulitan item $i$ (dalam logit)
-   $P(\cdot)$ = probabilitas menjawab benar

::: {.callout-note title="Hubungan 1PL dengan Rasch Model"}
Model 1PL dan Rasch Model secara matematis **identik** ketika konstanta *D* = 1. Perbedaan filosofis ada pada pendekatan:

|                  | 1PL (dalam IRT)          | Rasch Model               |
|------------------|--------------------------|---------------------------|
| Orientasi        | Fit model ke data        | Data harus fit ke model   |
| Estimasi         | MML (mirt)               | CML (eRm) atau JML (TAM)  |
| Perlakuan misfit | Item direvisi jika perlu | Item *harus* fit ke model |
:::

## Item Information Function (IIF) — Model 1PL

Untuk model 1PL, IIF memiliki bentuk:

$$I_i(\theta) = P_i(\theta) \cdot [1 - P_i(\theta)]$$

Informasi maksimum terjadi ketika $\theta = b$, yaitu:

$$I_i^{max} = 0.5 \times 0.5 = 0.25$$

::: {.callout-important title="Implikasi Informasi Maksimum"}
Karena semua item 1PL memiliki *a* = 1, **semua item memberikan informasi maksimum yang sama** (0.25) — hanya berbeda *di mana* informasi tersebut paling tinggi (yaitu pada θ = b masing-masing item).
:::

  nrow = n_peserta,
  dimnames = list(
    paste0("P", 1:n_peserta),
    paste0("Item", sprintf("%02d", 1:n_item))
  )
)

# ESTIMASI DENGAN MIRT (itemtype = "Rasch")
# ======================================================
model_1pl <- mirt(
  data     = data_1pl,
  model    = 1,
  itemtype = "Rasch",   # 1PL / Rasch

-   Titik biru (CML) dan segitiga oranye (MML) saling **tumpang tindih** di sepanjang garis. Artinya dalam konteks Model 1PL, pilihan metode estimasi tidak banyak membuat perbedaan hasil.

3.  **Akurasi konsisten di seluruh rentang b**

-   Dari b = -2.5 hingga b = +2.5, tidak ada area yang estimasinya lebih buruk. Artinya model 1PL bekerja stabil baik untuk soal mudah maupun soal sulit.

## Estimasi Kemampuan Peserta (θ)

```{r}
#| label: fig-theta-1pl
#| fig-cap: "Distribusi Estimasi Kemampuan Peserta — Model 1PL"

theta_est <- mirt::fscores(model_1pl, method = "EAP")[, 1]

df_theta <- data.frame(
  theta_true = theta_true,
  theta_est  = theta_est
)

cor_theta <- cor(theta_true, theta_est)

p1 <- ggplot(df_theta, aes(x = theta_est)) +
  geom_histogram(aes(y = after_stat(density)), bins = 30,
                 fill = "#3498db", alpha = 0.7, color = "white") +
  geom_density(linewidth = 1.2, color = "#1a5276") +
  geom_vline(xintercept = mean(theta_est), 
             linetype = "dashed", color = "#e74c3c", linewidth = 1) +
  labs(title = "Distribusi Estimasi θ (EAP)",
       subtitle = sprintf("Mean = %.3f, SD = %.3f",
                          mean(theta_est), sd(theta_est)),
       x = "θ Estimasi", y = "Densitas") +
  theme_minimal(base_size = 12)

p2 <- ggplot(df_theta, aes(x = theta_true, y = theta_est)) +
  geom_point(alpha = 0.4, size = 1.5, color = "#3498db") +
  geom_smooth(method = "lm", color = "#e74c3c", linewidth = 1, se = TRUE) +
  labs(title = "θ True vs θ Estimasi",
       subtitle = sprintf("r = %.3f", cor_theta),
       x = "θ True", y = "θ Estimasi (EAP)") +
  theme_minimal(base_size = 12)

library(patchwork)
p1 + p2
```

::: {.callout-important title="Apa yang Ditunjukkan Grafik Ini?"}
**Grafik Kiri: Distribusi Estimasi θ (EAP)**

**Apa yang Ditampilkan?**

Distribusi skor kemampuan seluruh peserta setelah diestimasi menggunakan metode **EAP (Expected A Posteriori)**.

**Membaca Angkanya**

| Statistik | Nilai  | Interpretasi                                    |
|-----------|--------|-------------------------------------------------|
| **Mean**  | -0.000 | Rata-rata kemampuan peserta tepat di tengah (0) |
| **SD**    | 0.906  | Sebaran kemampuan peserta cukup bervariasi      |

**Bentuk Distribusinya**

Kurva berbentuk **lonceng simetris (normal)** dengan:

-   Puncak tepat di θ = 0

-   Garis merah putus-putus (mean) persis di tengah

-   Ekor kiri dan kanan seimbang

**Artinya:** Sampel peserta terdistribusi normal — ada yang berkemampuan rendah, sedang, dan tinggi secara proporsional. Ini kondisi ideal untuk kalibrasi item.\
\
\
**Grafik Kanan: θ True vs θ Estimasi (r = 0.863)**

**Apa yang Ditampilkan?**

Perbandingan antara kemampuan **sesungguhnya** (true θ, sumbu X) dengan kemampuan **yang berhasil diestimasi** model (estimasi θ, sumbu Y).

**Membaca Korelasi r = 0.863**

```         
r = 0.863  →  korelasi KUAT ✅

Artinya: 86.3% variasi kemampuan sejati
berhasil "ditangkap" oleh estimasi model

Atau: R² = 0.745 → 74.5% variansi true θ
dijelaskan oleh θ estimasi
```

**Pola Scatterplot** Yang bagus:

-   Titik-titik mengikuti garis diagonal merah dengan rapat
-   Pola linear yang jelas dari kiri bawah ke kanan atas
-   Tidak ada pola melengkung (non-linear)

Yang perlu diperhatikan:

-   Di area θ ekstrem (\< -2 atau \> +2), titik mulai menyebar lebih lebar dari garis
-   Ini wajar — estimasi kurang presisi di area kemampuan ekstrem karena informasi tes lebih sedikit di sana (ingat kurva TIF!)

**Membaca Keduanya Secara Bersama**

```         
Grafik Kiri              Grafik Kanan
(Distribusi)             (Akurasi)
     +                        +
Distribusi normal    ×   r = 0.863
Mean ≈ 0                 Korelasi kuat
SD = 0.906
     ↓                        ↓
Sampel representatif  +  Estimasi akurat
     
     = Model 1PL bekerja dengan BAIK ✅
```

**Kesimpulan Evaluasi**

| Aspek             | Hasil              | Status               |
|-------------------|--------------------|----------------------|
| Distribusi θ      | Normal, Mean ≈ 0   | ✅ Ideal             |
| Akurasi estimasi  | r = 0.863          | ✅ Kuat              |
| Konsistensi model | Pola linear bersih | ✅ Baik              |
| Area ekstrem      | Sedikit menyebar   | ⚠️ Wajar, perhatikan |

>  **Kesimpulan:** Model 1PL berhasil mengestimasi kemampuan peserta dengan akurasi yang baik. Korelasi 0.863 menunjukkan bahwa meskipun tidak sempurna, model mampu **meranking peserta dengan tepat** — siapa yang benar-benar mampu dan siapa yang tidak.
:::
